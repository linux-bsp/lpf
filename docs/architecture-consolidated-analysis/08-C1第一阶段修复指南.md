# C-1 第一阶段修复实施指南

## 目标

修改Serial/I2C/SPI的pthread_mutex为OSAL_Mutex，实现HAL层的可移植性。

---

## 修改模式

### 1. 类型替换

```c
// 之前
pthread_mutex_t mutex;

// 之后
osal_mutex_t *mutex;
```

### 2. 初始化替换

```c
// 之前
pthread_mutex_init(&mutex, NULL);

// 之后
OSAL_MutexCreate(&mutex);
```

### 3. 加锁替换

```c
// 之前
pthread_mutex_lock(&mutex);

// 之后
OSAL_MutexLock(mutex);
```

### 4. 解锁替换

```c
// 之前
pthread_mutex_unlock(&mutex);

// 之后
OSAL_MutexUnlock(mutex);
```

### 5. 销毁替换

```c
// 之前
pthread_mutex_destroy(&mutex);

// 之后
OSAL_MutexDelete(mutex);
```

## 注意事项

### 1. 静态初始化问题

**问题**: pthread_mutex可以静态初始化
```c
pthread_mutex_t mutex = PTHREAD_MUTEX_INITIALIZER;
```

**解决**: OSAL_Mutex必须动态创建
```c
osal_mutex_t *mutex = NULL;

// 在Init函数中
OSAL_MutexCreate(&mutex);

// 在Deinit函数中
OSAL_MutexDelete(mutex);
```

### 2. 错误处理

**确保检查返回值**:
```c
int32_t ret = OSAL_MutexCreate(&ctx->mutex);
if (ret != OSAL_SUCCESS) {
    LOG_ERROR("HAL", "Failed to create mutex: %d", ret);
    // 清理资源
    return ret;
}
```

### 3. 资源清理

**确保在错误路径中清理**:
```c
int32_t HAL_Serial_Init(...) {
    ret = OSAL_MutexCreate(&ctx->mutex);
    if (ret != OSAL_SUCCESS) {
        goto error;
    }
    
    // ... 其他初始化
    
    return OSAL_SUCCESS;

error:
    if (ctx->mutex) {
        OSAL_MutexDelete(ctx->mutex);
    }
    return ret;
}
```

### 4. 线程安全

**保持原有的线程安全逻辑**:
- 不要改变加锁的范围
- 不要改变加锁的顺序
- 保持原有的错误处理
