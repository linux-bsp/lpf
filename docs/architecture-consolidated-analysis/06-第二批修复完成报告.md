### 1. C-2: PCL依赖PDL形成循环依赖 ✅

**修复方案**：
- 创建独立的`core/types`模块
- 将PDL配置类型定义提取到types模块
- 打破循环依赖

**架构变更**：
```
之前: PCL ←→ PDL (循环依赖)
之后: PCL → types ← PDL, PDL → PCL (清晰的依赖链)
```

**新增文件**：
- `core/types/include/pdl_types.h` - PDL配置类型定义
- `core/types/CMakeLists.txt` - 构建配置
- `core/types/Kconfig` - 配置选项
- `core/types/config.cmake` - CMake配置

**修改文件**：17个文件

**验证结果**：

### 4. C-10: PCL全局状态无线程保护 ✅

**问题描述**：PCL全局注册表无互斥锁保护，多线程不安全

**修复方案**：
- 添加互斥锁保护全局注册表
- 在PCL_Init中创建互斥锁
- 在所有访问函数中添加锁保护

**关键改动**：
```c
static osal_mutex_t *g_registry_mutex = NULL;

int32_t PCL_Init(void) {
    ret = OSAL_MutexCreate(&g_registry_mutex);
    ...
}

int32_t PCL_Register(...) {
    OSAL_MutexLock(g_registry_mutex);
    // 注册逻辑
    OSAL_MutexUnlock(g_registry_mutex);
}
```

**新增测试**：
- `tests/unit/pcl/test_pcl_api.c` - 多线程单元测试（283行）

**修改文件**：2个文件

**验证结果**：
- ✅ 所有全局状态访问有锁保护
- ✅ 多线程单元测试通过
- ✅ 编译成功
