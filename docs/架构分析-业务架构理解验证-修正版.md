5. ✅ **配置驱动**：通过 ACL 和 PCL 配置实现业务与硬件解耦

## 📋 完整架构总结

### 模块职责

| 模块 | 全称 | 职责 | 依赖 |
|------|------|------|------|
| **OSAL** | Operating System Abstraction Layer | 操作系统抽象 | 无 |
| **HAL** | Hardware Abstraction Layer | 硬件抽象 | OSAL |
| **PCL** | Peripheral Configuration Library | 外设硬件配置（串口号、波特率等） | OSAL（仅类型定义） |
| **PRL** | Protocol Layer | 协议层（编解码） | **OSAL（必须且只能）** |
| **PDL** | Peripheral Driver Layer | 外设驱动层 | PRL, HAL, OSAL, PCL |
| **ACL** | Application Configuration Layer | 业务配置（功能→外设映射） | OSAL, PDL |

### 依赖关系图

```
Apps (应用层)
  ↓
ACL (业务配置层：功能→外设映射)
  ↓
PDL (外设驱动层)
  ├─→ PRL (协议层：只依赖 OSAL) ─→ OSAL
  ├─→ HAL (硬件抽象层) ─→ OSAL
  ├─→ PCL (硬件配置层：串口号、波特率等)
  └─→ OSAL (操作系统抽象层)
```

### 协议层架构

```
┌─────────────────────────────────────────────────────────┐
│                    PRL (协议层)                          │
│  - 只依赖 OSAL，不依赖标准库                             │
│  - 支持跨平台（Linux 和 RTOS）                           │
│  - 传输层无关（TCP/UDP/CAN/UART/I2C/SPI）                │
├─────────────────────────────────────────────────────────┤
│  PRL_GSC_PMC      │ 地面站 ↔ PMC                        │
│  PRL_PMC_CCM      │ PMC ↔ CCM                           │
│  PRL_CCM_Satellite│ CCM ↔ 卫星平台                      │
│  PRL_CCM_Power    │ CCM ↔ 电源板                        │
│  PRL_MCU          │ 多个设备 ↔ MCU（新增）              │
└─────────────────────────────────────────────────────────┘
```

### 配置层架构

```
┌─────────────────────────────────────────────────────────┐
│                 ACL (业务配置层)                         │
│  - 业务功能 → 物理外设的逻辑映射                         │
│  - 例如：遥测功能使用第 2 个 MCU                         │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                 PCL (硬件配置层)                         │
│  - 物理外设的硬件接口特性                                │
│  - 例如：第 2 个 MCU 使用 /dev/ttyS1，波特率 115200     │
└─────────────────────────────────────────────────────────┘
```

### 扩展性设计

**当前通信链路**：
- GSC ↔ PMC
- PMC ↔ CCM
- CCM ↔ Satellite
- CCM ↔ Power
- CCM → BMC
- CCM ↔ MCU

**未来可能的扩展**：
- GSC ↔ CCM（地面站直接与 CCM 通信）
- GSC ↔ Satellite（地面站直接与卫星平台通信）
- PMC ↔ BMC（带内管理访问带外管理）
- PMC ↔ MCU（PMC 也可能有 MCU 外设）
- CCM ↔ CCM（多个 CCM 之间互联）
- Satellite ↔ BMC（卫星平台与 BMC 通信）

**扩展方式**：
1. 协议层：复用已有 PRL 协议
2. 传输层：通过 HAL 适配不同传输介质
3. 配置层：通过 ACL 和 PCL 配置新的通信链路
4. 无需修改协议层代码

### 3. 重构 PDL_Satellite 使用 PRL

**修改文件**：`core/pdl/src/pdl_satellite/pdl_satellite.c`

```c
#include "prl_ccm_satellite.h"  // 添加 PRL 头文件

int32_t PDL_Satellite_SendHeartbeat(pdl_satellite_handle_t handle,
                                     pdl_satellite_status_t status) {
    pdl_satellite_context_t *ctx = (pdl_satellite_context_t *)handle;
    
    // 准备 PRL 消息
    prl_ccm_satellite_heartbeat_t msg = {
        .ccm_status = status,
        .satellite_status = 0,
        .link_status = 1,
        .error_count = ctx->error_count
    };
    
    // 调用 PRL 编码
    uint8_t buf[PRL_MAX_PACKET_SIZE];
    size_t len = sizeof(buf);
    int ret = prl_ccm_satellite_encode_heartbeat(&msg, buf, &len);
    if (ret != PRL_OK) {
        return OSAL_ERR_GENERIC;
    }
    
    // 通过 HAL 发送（传输层可变）
    return HAL_CAN_Write(ctx->can_handle, buf, len);
}
```
